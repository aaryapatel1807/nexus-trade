## 📋 Deployment Steps

### Option A: Deploy via Render Dashboard (Recommended)

1. **Go to render.com and sign in**

2. **Create database (one-time only)**
   - Click "Create Resource" → "PostgreSQL"
   - Name: `nexus-trade-db`
   - Plan: Free
   - Region: Oregon (or your preferred region)
   - Click "Create Database"
   - Copy the **Internal Database URL** for later

3. **Deploy backend**
   - Click "Create Resource" → "Web Service"
   - Connect your GitHub repository
   - Name: `nexus-trade-backend`
   - Root Directory: `backend`
   - Build Command: Set to `npm ci && npx prisma generate`
   - Start Command: Set to `npx prisma migrate deploy && node server.js`
   - Runtime: Node
   - Plan: Free
   - **Environment Variables** tab:
     - `DATABASE_URL` = (get from database service)
     - `JWT_SECRET` = (let Render generate)
     - `NODE_ENV` = `production`
     - `PORT` = `5000`
     - `GEMINI_API_KEY` = (paste your actual key)
     - `FRONTEND_URL` = (will auto-populate after frontend deploys)

4. **Deploy frontend**
   - Click "Create Resource" → "Static Site"
   - Connect your GitHub repository
   - Name: `nexus-trade-frontend`
   - Root Directory: `frontend`
   - Build Command: Set to `npm ci && npm run build`
   - Publish Directory: `dist`
   - **Environment Variables** tab:
     - `VITE_API_URL` = `https://nexus-trade-backend.onrender.com` (or your backend service URL)

### Option B: Deploy via render.yaml (Automated)

1. **Push render.yaml to GitHub**
   ```bash
   git add render.yaml
   git commit -m "chore: update render.yaml with correct deployment config"
   git push
   ```

2. **Go to render.com → Dashboard**

3. **Create new service from blueprint**
   - Click "Create Resource" → "Blueprint"
   - Select your GitHub repository
   - Select branch: `main`
   - Click "Create from Blueprint"

4. **Render will deploy all services from render.yaml automatically**

## 🐛 Troubleshooting

### Problem: `CORS error` when accessing from frontend
**Solution**: 
- Check `FRONTEND_URL` is set correctly in backend environment
- Verify frontend service host in Render dashboard
- May take 5-10 minutes after deployment for settings to apply

### Problem: `JWT_SECRET is required` error
**Solution**:
- JWT_SECRET should be auto-generated by Render
- If blank, manually set it:
  - `openssl rand -base64 32` (generate new)
  - Add to backend environment variables

### Problem: Database connection fails
**Solution**:
- Check `DATABASE_URL` is set and valid
- Verify database service is running (green status)
- Try `npx prisma migrate deploy --skip-generate` manually
- Check logs for SSL/connection errors

### Problem: Frontend shows blank page
**Solution**:
- Check `VITE_API_URL` is set correctly
- Should be `https://nexus-trade-backend.onrender.com`
- Force refresh browser (Ctrl+Shift+R)
- Check browser console and Render logs for errors

### Problem: Stock prices showing as 0 or 50000
**Solution**:
- This was fixed in the price mismatch commit
- If still seeing old behavior:
  - Rebuild frontend: Backend logs → "Rebuilding"
  - Or manually trigger rebuild in Render Dashboard
  - Wait 5-10 minutes for cache to clear

### Problem: `npm install` fails during build
**Solution**:
- Use `npm ci` instead of `npm install`
- This uses package-lock.json (consistent installs)
- Already configured in render.yaml

## 📈 Scaling Up From Free Plan

When you hit free plan limits:

**Backend Memory**: Free tier → Starter tier ($7/month)
**Database**: Free tier → Starter tier ($15/month)
**Storage**: Free → 100GB ($0.15/GB)

To upgrade:
1. Go to service settings
2. Click "Upgrade Plan"
3. Select new tier
4. Scale will restart service (5-10 sec downtime)

## 🔄 Continuous Deployment

Once deployed, every `git push` to `main` triggers:
1. Render detects changes
2. Rebuilds backend service
3. Rebuilds frontend service
4. Zero-downtime deploy (if no DB changes)

To prevent auto-deploy:
- Go to service settings
- Disable "Auto-Deploy"
